# Contributing

Version: v1.0.0  
Status: Initial governance

Thank you for contributing to Fava Imen Olka repositories.

## Principles

- Use professional English.
- Follow the Enterprise Manual and engineering standards.
- Keep changes focused.
- Do not overclaim product status or company capability.
- Do not commit secrets, credentials, private customer data, or production data.
- Update documentation when behavior, architecture, standards, or strategy changes.

## Workflow

1. Review the relevant documentation and standards.
2. Create a focused branch.
3. Make the smallest complete change that solves the problem.
4. Add or update tests where applicable.
5. Update documentation.
6. Open a pull request with verification notes and risks.

## Documentation Contributions

- Prefer improving existing documents over creating duplicates.
- Preserve historical knowledge.
- Clearly separate completed, in-progress, research, and planned work.
- Add decisions to `decisions/` and architecture decisions to `adr/` when needed.

## Code Contributions

- Follow `standards/coding-standards.md`.
- Follow `standards/security-standards.md`.
- Keep business logic, infrastructure, and presentation concerns separated where practical.
- Include tests proportional to risk.

## Review Expectations

Pull requests are reviewed for correctness, security, maintainability, documentation quality, operational impact, and alignment with company standards.
