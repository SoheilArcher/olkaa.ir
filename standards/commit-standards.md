# Commit Standards

Version: v1.0.0  
Status: Initial standard

## Commit Message Format

Use clear, focused commit messages.

Recommended format:

```text
type: short description
```

Common types:

- `docs`
- `feat`
- `fix`
- `refactor`
- `test`
- `chore`
- `security`

## Rules

- Use the imperative mood where practical.
- Keep the subject concise.
- Do not combine unrelated changes.
- Mention documentation-only work with `docs`.
- Avoid vague messages such as `update`, `changes`, or `fix stuff`.

## Examples

```text
docs: establish engineering standards
feat: add customer status dashboard
fix: validate invoice payment status
security: restrict admin export permissions
```
