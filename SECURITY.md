# Security Policy

Version: v1.0.0  
Status: Initial governance

Fava Imen Olka treats security as a default engineering requirement.

## Reporting A Vulnerability

Do not open public issues for sensitive vulnerabilities.

Report security concerns through the approved internal or private reporting channel. If a public repository is used, contact the repository owner privately before sharing details.

## What To Include

- Affected repository, product, or document.
- Vulnerability summary.
- Steps to reproduce.
- Potential impact.
- Evidence such as logs or screenshots, with secrets removed.
- Suggested mitigation if known.

## Handling Rules

- Do not include secrets, credentials, tokens, private keys, customer data, or production data in reports.
- Do not exploit vulnerabilities beyond what is necessary to demonstrate impact.
- Do not access, modify, or delete data without authorization.
- Security fixes should include documentation and verification notes.
