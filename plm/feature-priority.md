# Feature Priority

Version: v1.0.0  
Status: Template

This document defines how features are prioritized.

## Priority Levels

- Critical: Required for security, production stability, legal obligation, or major customer impact.
- High: Strong business value or important roadmap dependency.
- Medium: Useful improvement with clear value but not urgent.
- Low: Nice-to-have improvement or future consideration.

## Prioritization Rules

- High value does not automatically override security or operational risk.
- Features must align with product lifecycle stage.
- Roadmap alignment should be documented.
- Rejected or deferred features should keep the reason for future reference.
