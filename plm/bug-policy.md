## Severity Levels

- Critical: Production outage, data loss, security vulnerability, or major customer impact.
- High: Important workflow broken with limited workaround.
- Medium: Incorrect behavior with workaround.
- Low: Minor issue, cosmetic defect, or documentation inconsistency.

## Status Values

- New
- Confirmed
- In Progress
- Fixed
- Verified
- Closed
- Rejected
- Duplicate

## Rules

- Security bugs must be handled privately.
- Production bugs require impact and workaround notes.
- Bug fixes should include regression tests where practical.
- Closed bugs should include verification notes.
