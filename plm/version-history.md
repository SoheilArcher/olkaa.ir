## Versioning Rules

- Use semantic versioning where applicable.
- Major versions represent breaking changes or major product milestones.
- Minor versions represent feature additions or meaningful improvements.
- Patch versions represent fixes, small improvements, and documentation updates.
- Production versions must have release notes and support ownership.
