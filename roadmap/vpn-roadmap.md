# Vision

Create a controlled accounting and subscription-management product for VPN and connectivity services, connecting customer records, plans, billing, renewals, access status, support notes, and reports.

# Current Stage

VPN Accounting is Planned. Requirements, integration boundaries, security controls, and billing workflows still need approval.

# Version History

- v1.0.0: Initial roadmap created.

# v1.0

- Define customer, plan, subscription, invoice, and payment data models.
- Document access-control boundaries between billing and network operations.
- Define renewal workflow and notification requirements.
- Identify reporting needs for revenue and service status.
- Create security and audit-log requirements.

# v1.1

- Add payment gateway and accounting integration specifications.
- Add service activation and suspension workflow design.
- Add customer support and service-note workflows.
- Prepare MVP implementation checklist.
- Define backup and recovery requirements.

# v2.0

- Add advanced reporting and operational dashboards.
- Add controlled integration with VPN or network systems if approved.
- Add customer portal capabilities where appropriate.
- Add lifecycle automation with human review for sensitive actions.

# Future Ideas

- AI-assisted renewal-risk analysis.
- Automated support summaries.
- Billing anomaly detection.
- Natural-language revenue and subscription reporting.
- Integration with NetLog for service visibility.

# Risks

- Billing errors may affect customer trust.
- Network-access automation may create security risk if poorly controlled.
- Payment and personal data require careful protection.
- Scope may drift into a full ISP platform before MVP validation.

# Dependencies

- Accounting standards.
- Payment gateway decisions.
- Network integration requirements.
- Security and audit-log standards.
- Customer data protection rules.

# Success Metrics

- Billing workflows are documented and reviewed.
- Subscription status is auditable.
- Access changes require appropriate authorization.
- Reports reconcile with accounting records.
- MVP does not automate risky network actions before controls exist.
