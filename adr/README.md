# Architecture Decision Records

Version: v1.0.0  
Status: Initial structure

This folder stores Architecture Decision Records (ADRs) for Fava Imen Olka systems and products.

## Naming

Use sequential ADR files:

- `ADR-0001.md`

## Template

```markdown
# ADR-0001: Title

## Title

Short architecture decision title.

## Status

Proposed | Accepted | Superseded | Deprecated

## Context

Describe the technical context, constraints, requirements, and forces.

## Decision

Document the architecture decision.

## Consequences

Describe positive outcomes, tradeoffs, risks, and required follow-up.

## Alternatives

List the alternatives considered and why they were not selected.
```

## Rules

- Create ADRs for meaningful architecture choices.
- Prefer improving or superseding existing ADRs over creating duplicate records.
- Keep ADRs factual and linked to related products or systems where useful.
