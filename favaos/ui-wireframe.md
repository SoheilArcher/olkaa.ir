# FavaOS UI Wireframe

Version: v1.0.0  
Status: Layout design only

This document describes the user interface layout. It does not include HTML or application code.

## Overall Layout

- Left sidebar for primary navigation.
- Top bar for page title, search, current user, notifications, and quick actions.
- Main content area for dashboards, tables, forms, reports, and detail views.
- Optional right panel for context, related documents, risks, and decisions.

## Executive Dashboard Layout

Top area:

- Company health summary.
- Current month goals.
- Decisions needed.
- Highest priority risks.

Middle area:

- Product health.
- Project progress.
- Customer status.
- Revenue indicators.
- Roadmap progress.

Lower area:

- Documentation coverage.
- Technical debt.
- AI usage.
- Infrastructure status.
- Recent decisions and ADRs.

## List Page Layout

Each module list page should include:

- Page title.
- Primary action button.
- Search.
- Filters.
- Status tabs.
- Data table.
- Bulk actions where appropriate.
- Export option where permission allows.

## Detail Page Layout

Each detail page should include:

- Record title.
- Status badge.
- Owner.
- Key fields.
- Related products, projects, customers, contracts, risks, documents, decisions, and ADRs.
- Activity history.
- Audit events where applicable.

## Form Layout

Forms should include:

- Clear section headings.
- Required field indicators.
- Help text for sensitive or strategic fields.
- Validation messages.
- Save, cancel, and review actions.

## Report Layout

Reports should include:

- Reporting period.
- Executive summary.
- Metrics.
- Risks.
- Decisions needed.
- Follow-up actions.
- Links to source records.

## UI Rules

- Use professional, quiet, operational styling.
- Avoid decorative layouts that reduce information density.
- Keep status labels consistent.
- Make dashboards scannable.
- Make ownership and next action visible.
